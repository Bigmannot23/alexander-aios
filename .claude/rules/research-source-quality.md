# Research Source Quality

Before treating any external source as usable input, capture:

- Source type (article, video, course, transcript, forum post, paper, etc.)
- Date (published/recorded — not date-ingested)
- Author/provider
- Category: primary, secondary, marketing, or speculation — don't blur these
- Claim confidence for any factual/legal/financial/performance claim, via
  `Templates/claim-audit.md`
- Rights/IP gate (green/amber/red) before any promotion — see blueprint §7.2
- Never store raw transcript, video, or audio in this repo — extract, don't
  archive
- In any output, keep four buckets separate and labeled: facts,
  recommendations (the source's, not Claude's), assumptions, and open
  questions

Full reasoning: `Master-Blueprint-V1.md` §7.
